# Support

Atlas is pre-release, volunteer-maintained software. Support is best-effort and
does not include guaranteed response or resolution times.

## Where to ask

- [Report a reproducible bug](https://github.com/xkam7ar/atlas-download-manager/issues/new?template=bug_report.yml).
- [Request a focused feature](https://github.com/xkam7ar/atlas-download-manager/issues/new?template=feature_request.yml).
- [Ask a usage question or discuss an idea](https://github.com/xkam7ar/atlas-download-manager/discussions).
- For substantial code changes, read `CONTRIBUTING.md` before opening a pull
  request.
- Report security vulnerabilities privately according to `SECURITY.md`.

Before opening an issue, search existing issues, reproduce with the latest
`main` branch or latest supported release, and collect the Atlas, operating
system, Python, and relevant downloader versions.

## Protect private information

Never post authentication cookies, passwords, tokens, private URLs, personal
paths, unredacted configuration, or copyrighted content. Reduce logs to the
smallest relevant excerpt and sanitize them before sharing.

## Support boundaries

Maintainers cannot provide legal advice, account-recovery help, access to
third-party services, or assistance bypassing access controls. Atlas must be
used only for content and systems you are authorized to access. Questions about
third-party downloader behavior may need to be raised with that upstream
project.
